# Critical Patterns - Required Reading

**Purpose:** This document contains critical patterns that ALL agents must follow. These patterns have been identified from repeated issues across multiple components.

**When to add here:** Only add patterns that:
1. Have caused the same mistake multiple times
2. Have non-obvious solutions that must be followed every time
3. Are foundational requirements (framework, threading, async patterns)

---

## How to Use This Document

1. **Before code generation:** Check if any pattern applies to your current task
2. **During review:** Verify implementations follow these patterns
3. **After solving issues:** Consider if the solution should become a pattern (use `/omnipowers:compound` option 2)
